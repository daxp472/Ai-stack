# Contributing to AI-Stack-by-Dax

Thank you for helping improve this repository. We want this to stay practical, high-signal, and creator-friendly.

## What You Can Contribute
- New AI tools that are actively useful
- Better usage tips from real projects
- Strong prompt templates with clear outcomes
- Fixes for outdated links or tool descriptions

## Contribution Rules
1. Keep entries concise and evidence-based.
2. Avoid duplicate tools unless the use case is clearly different.
3. Add at least 2 tested prompts when adding a new tool.
4. Use clean formatting and follow existing folder conventions.

## Suggested Workflow
1. Fork the repository.
2. Create a branch:
   ```bash
   git checkout -b feat/add-new-tool
   ```
3. Update the relevant category `README.md` and `best-prompts.md`.
4. Commit with a clear message:
   ```bash
   git commit -m "Add [ToolName] to [Category] with tested prompts"
   ```
5. Open a pull request with before/after value.

## Pull Request Checklist
- [ ] Tool is relevant to category
- [ ] Description is practical and short
- [ ] Prompts are copy-paste ready
- [ ] Links are valid
- [ ] Formatting matches repo style

> [!TIP]
> The best contributions include a mini example of output quality so others can quickly trust and adopt the tool.

Thanks for building this with us. 🚀
